我的2016

2016

今天是2016年的最后一天，是周末，嗯，也是我的农历生日。总是想抽空写点什么，但是今年一年似乎都追逐在让自己能力提升的路上，并没有静下心来写点东西。事实上，个人的提升并没有达到我预期的结果。

昨天，同事分享了一篇文章，心有戚戚焉！眼看今天又年长了一岁，距离30岁更近了一步，心中不禁惶恐。不知道是不是因为年纪大了，感觉更加的迷茫了，一直在寻找方向与出路。这篇文章给出了技术人员几种不同的发展方向，但是有一个前提是自己很牛逼，能展现出Leadership特质。感觉，所有的鸡汤文都可以用一句话来总结：你只有自己很牛逼才会成功，才有更多的选择。

今年这一年，可能是我30岁以前进步最快的一年。当然，不一定是指技术水平进步最快，而是眼界更远了。这得益于51offer的同事们都非常优秀，与这些很优秀的人共事，会逼着自己往前跑。每周技术分享，让自己提升了表达能力，更加深了自己对某一知识点的掌握程度。每个月OKR，能让自己做自己想做的事情并为公司做出贡献，虽然完成度不高，但是确实能让自己进步。在以前的开发工作过程中，我都是个人开发，以需求为中心，只顾按时完成与交付。现在多人的开发，让自己学会各种技能，学习其他人的代码，了解其他人的代码风格，提升沟通能力，这些自不多说。

阅读

生活所迫，上下班路途遥远，且高峰期上海开车限行。唯一的好处是有一整片的时间用来阅读。阅读微信公众号的文章、微信读书、多看阅读等，受益匪浅。这一年的阅读书单：

  • 《黑客与画家》
    • 把程序员等同于画家、建筑设计师等创作型职业类比非常有意思，同时格雷厄姆的创业公式也总结的非常的准确，也适用于现在的创业模式。梦寐以求的编程语言这一章节尤其触动人心：为了写出优秀的软件，必须具有两种互相冲突的信念：千难万难只等闲和早岁那知世事艰。也就是说既要有信心也要持有怀疑的态度。
  • 《从0到1开启商业与未来的秘密》
    • 讲述了公司从0到1的过程，两种创新模式：复制（从1到n）和创造财富（从0到1），里面的观点非常值得仔细研究。
  • 《MacTalk-人生元编程》
  • 《MacTalk-跨越边界》
    • MacTalk这两本书都是整理自他的微信公众号文章。作为鸡汤文的形式阅读，聊程序员、聊新产品、聊公司等，比较符合我的口味。
  • 《把时间当做朋友》
    • 当读完这本书的时候，我决定把李笑来当做我的偶像。然后萌生出一个App idea：做一个时间管理App，方便的记录自己每天的时间开销。没过多久看到App Store首页有推荐一个时间块的App实现了我的这个想法，也就没有再继续下去了。读完这本书以后，对我最大的影响的是：当我懒惰、懈怠、安于舒适区的时候，我会对自己说，别让大脑控制了自己！哈哈
  • 《逻辑思维：迷茫时代的明白人》
    • 第一次知道老罗是在微信公众号里，很佩服他能每天早上6点钟发一条语音消息。坚持了这么牛也该成功了吧！
  • 《未来边缘》
    • 比起《三体》，这本书不是讲一个故事，而是讲了几个科幻故事，依旧让人脑洞打开。不得不佩服大刘超凡的想象力。
  • 《活着》
    • 这本书名叫活着，写的却是死亡。我们这一代人无法体会到爷爷辈那种艰苦的岁月，但这本书非常生动的以第一人称讲述了死亡的故事，让人萧然泪下。现在生活中的这点挫折，比起那个年代简直屁都不是！
  • 《偷影子的人》
    • 很美好的故事。关于孤独、关于爱情、关于母爱、关于现实都有很好的体现。初读此书以为是国人写的，可见译者的翻译功底的厉害。
  • 《乖，摸摸头》
    • 不能像大冰那样周游世界，体验各种不一样的生活，不能拥有各种生活圈，总可以看看他们的故事吧！文章有暖心的爱情故事，战场的铁血故事，兄弟的奋斗故事，梦想的追求故事，让人回味。
  • 《你只是看起来很努力》
    • 感觉李尚龙的这本书是标题党，书中描述的故事大多都不深刻。
  • 《成为乔布斯》
    • 这本书讲述了乔布斯的成长过程。很生动的讲了乔布斯的性格、生活、谈判技巧、职业生涯以及缺陷，到最后成为人们心目中所想的乔布斯。
  • 《史蒂夫·乔布斯传》
    • 我是作为成为乔布斯的补充阅读的
  • 《软技能·代码之外的生存指南》
    • 这本书是今年对我工作、生活以及态度影响最大的一本书！像经营企业一样经营自己改变我对作为一名雇员的理解；打造自己的品牌让我意识到自我营销的重要性；十步学习法让我拥有了很快学习新技术的方法论；专注和番茄工作法让我可以提升自己的生产力；理财知识让我明白期权和投资的概念及意义；健身对于程序员的重要性；怎样规划自己的大脑从心灵影响到行动。每一章节都有具体的采取行动的方法，阅读此书简直是如获至宝！
  • 《习惯的力量》
    • 用大量的实践和实例证明描述了习惯回路和习惯是怎么工作的，怎样改变一个习惯以及习惯的力量。阅读此书，让人坚信找到渴求感后，在暗示和奖赏不变的情况下，通过改变惯常行为是可以改变一个人的习惯的。在以后的生活中，希望自己通过这一理论可以改变坏习惯，创建新的习惯，利用习惯的力量让自己牛逼起来。
  • 《简约至上》
    • 本书最大的收获是：不要为专家用户而设计，而是要为主流用户设计！简约四策略：删除、组织、隐藏和转移。

什么是CloudKit

  • iCloud
  • Documents
  • iCloud Drive
  • CloudKit

CloudKit 基础对象类型

  • CKContainer: Containers 就像应用运行的沙盒一样，一个应用只能访问自己沙盒中的内容而不能访问其他应用的。Containers 就是最外层容器，每个应用有且仅有一个属于自己的 container。

  • CKDatabase: Database 即数据库，私有数据库用来存储敏感信息，比如说用户的性别年龄等，用户只能访问自己的私有数据库。应用也有一个公开的数据库来存储公共信息，例如你在构建一个基于地理位置的应用，那么地理位置信息就应该存储在公共数据库里以便所有用户都能访问到。

  • CKRecord: 即数据库中的一条数据记录。CloudKit 使用 record 通过 key-value 结构来存储结构化数据。关于键值存储，目前值的架构支持 NSString、NSNumber、NSData、NSDate、CLLocation，和 CKReference、CKAsset（这两个下面我们会说明），以及存储以上数据类型的数组。

  • CKRecordZone: Record 不是以零散的方式存在于 database 之中的，它们位于 record zones 里。每个应用都有一个 default record zone，你也可以有自定义的 record zone。

  • CKRecordIdentifier: 是一条 record 的唯一标识，用于确定该 record 在数据库中的唯一位置。

  • CKReference: Reference 一种引用关系。以地理位置签到应用为例，每个地理位置可以包含很多用户在该位置的签到，那么位置与签到之间就形成了这样一种包含式的从属关系。

  • CKAsset: 即资源文件，例如二进制文件。还是以签到应用为例，用户签到时可能还包含一张照片，那么这张照片就会以 asset 形式存储起来。

JSPatch初探

发表于 2016-04-29 |

什么是JSPatch

JSPatch是一个开源项目，最初是腾讯@bang的个人项目，诞生于2015年5月。它能够使用JavaScript调用Objective-C的原生接口，从而动态植入代码来替换旧代码，以实现修复线上bug。

JSPatch平台

JSPatch 需要使用者有一个后台可以下发和管理脚本，并且需要处理传输安全等部署工作。如果自己搭建后台下发 JSPatch 脚本，可以直接使用 github 上的代码。JSPatch 平台提供了脚本后台托管，版本管理，保证传输安全等功能，无需搭建一个后台，无需关心部署操作，只需引入一个 SDK 即可立即使用 JSPatch。

Runtime简介及实践

发表于 2016-04-29 |

前言

今年年初的面试中，基本上超过了60%的公司会问RunTime的一些用法。
在很多开源项目中用到了RunTime的方法，使我们理解起来比较吃力。
RunTime确实很强大，借助它可以实现很酷的功能。比如JSPatch。
我本身对于RunTime的理解也没有很深入，所以这里讨论的主要是RunTime的具体应用场景。此番探索希望能起到抛砖引玉的作用，大家可以将RunTime应用起来。

简介

  • RunTime简称运行时。是一套底层的C语言API，包含很多强大实用的C语言数据类型和C语言函数，平时我们编写的ObjC代码，底层都是基于runtime实现的。ObjC就是运行时机制，也就是在运行时候的一些机制，其中最主要的是消息机制。比如：
  • 对于C语言，函数的调用在编译的时候会决定调用哪个函数。
  • 对于OC的函数，属于动态调用过程，在编译的时候并不能决定真正调用哪个函数，只有在真正运行的时候才会根据函数的名称找到对应的函数来调用。
  • 事实证明：
    • 在编译阶段，OC可以调用任何函数，即使这个函数并未实现，只要声明过就不会报错。
    • 在编译阶段，C语言调用未实现的函数就会报错。

那些一般人不会告诉你的提升做事效率的方法

发表于 2015-04-12 |

工作以来，就喜欢琢磨一些提升自己做事效率的方法，积累了一段时间，也看了很多相关的书籍与文章后，感到有了很大的收获，分享给各位，希望对大家有所帮助。

1、躺在床上的时候，不要玩手机。大学的时候喜欢在床上玩手机，结果经常会不知不觉玩到很晚，第二天经常会因为睡眠不足而精力不足。现在，躺在床上的时候就把手机放在一旁，睡眠效率提升很多。

2、永远不要在起床的时候看邮件，也永远不要在睡前看邮件。对我来说，我一般一天只在 3 个时间点去处理邮件，中午 11 点，下午 2 点，下午 6 点。此外，极其重要的一点是，你的邮箱的收件箱不应该是一个未完成事件列表，你应该要做到及时清空，如果是该做的任务（那就加入待办事项），如果是重要的文件（那就转存 Evernote 中或者存档），如果是无用文件（那就应该马上删除），千万不能把自己的邮箱弄的一团糟。

3、2 分钟法则：如果一件事情，在 2 分钟内可以处理完，那么就马上着手解决它。因为很多这样的事情，甚至没必要排到待做事项当中，越早解决，越节省时间。

4、5 分钟法则：治疗拖延症的最好方法，不是去设立各种各样的任务，也不是去下载各种各样的时间管理软件，而是告诉自己开始去尝试做个 5 分钟试试看就可以了。当你坐下来做个 5 分钟，你会发现你自己不知不觉就做下去了，很多事情没你想象的那么困难。所以，如果，你有什么一直想做却一直没做的事情的话，就坐下来做个 5 分钟试试看吧。
